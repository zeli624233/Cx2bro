# Cx2bro

适用于 **krkrz 引擎、CxdecV2 (hxv4)** 加密方式的 Galgame 解包工具。支持多种解包模式，可制作并分享游戏的 Cx2bro 扩展集。**扩展集库越丰富，一键解包的通用性越强**。

- 如果你喜欢的 Gal 是采用 krkrz 引擎、2022年12月底发售的游戏，那么本项目应该有用。
- 如果它确实帮到了你，麻烦给本项目一个 Star，这是对开发者最好的赞誉！

## 首页（三大解包功能）

### 前置操作
加载游戏的 `.exe` 主程序

> ⚠️ **重要注意事项**
> 1. 游戏不能包含密钥验证、通行证、加密校验类限制
> 2. 请勿将工具与游戏放置在 C 盘目录下
> 3. 建议使用全英文路径，规避未知异常问题

---

### 1. 该作扩展集提取（推荐）
- 步骤0：选择对应游戏的扩展集（扩展集库已收录方可使用）
- 步骤1：根据扩展集生成静态 Hash 表
- 步骤2：提取游戏动态 XP3 资源
- 步骤3：使用静态 Hash 表还原文件名称

> **补充说明**
> 解包文件自动输出目录：`游戏exe目录\user\1`，生成三个文件夹：
> - `Extractor_Output`：存放哈希格式原始文件
> - `StaticHash_Output`：存放扩展集生成的静态哈希数据
> - `Restored_Extractor_Output`：最终还原完毕的资源目录

### 2. 该会社集合撞新作（实验性功能）
- 步骤0：选择游戏所属会社（扩展集库收录样本充足时使用）
- 步骤1：根据会社合集生成静态 Hash 表
- 步骤2：提取游戏动态 XP3 资源
- 步骤3：使用静态 Hash 表还原文件名称

> 目前处于测试阶段，样本数量不足，还原效果图一乐。

### 3. 传统动态模式（兜底方案）
- 步骤1：提取游戏动态 XP3 资源
- 步骤2：加载动态 Hash 收集模块（C++ 原生窗口，实时监控）
- 步骤3：使用动态哈希数据还原原始文件名

> **补充说明**
> 前两种模式失效时使用。哈希收集需运行游戏（支持快进），支持断点续存。
> 解包文件自动输出目录：`游戏exe目录\user\3`

## 制作扩展集
扩展集为本工具核心功能，制作流程简易快捷：

1. 使用传统动态模式完成解包，保证文件还原率不低于75%
2. 生成当前游戏专属扩展集
3. 编辑扩展集信息（会社、作品名、贡献者、版本、日期、说明）
4. 自动校验扩展集可用性，统计还原成功率
5. 将扩展集安装至本地资源库
6. 调用专属扩展集模式复测解包

> **扩展集文件说明**
> 成品包含两份核心文件：
> - `manifest.int`：存储展示类信息
> - `rules.int`：二进制核心规则文件（CXRI3 格式，LZSS 压缩）
> ⚠️ 禁止使用记事本直接打开编辑，极易造成文件损坏

## 扩展集管理
- 统计查看软件扩展集总量、会社、作品与详细参数
- 支持扩展集测试、校验、本地目录快速跳转
- 🆕 **在线更新**：从 GitHub 仓库检查并下载新增扩展集，自动安装到本地库
- 9 列表格展示：入库编号、会社、作品、发售日、入库日、解包率、体积、贡献者、备注

> Cx2bro 扩展集是 Cx2bro 的核心功能，扩展集库越丰富，玩家解包游戏时就越方便。  
> 如果你成功还原了某个游戏的资源，不妨花 2 分钟制作扩展集并分享出来，帮助更多人。

## v1.2.0 新增功能
- **扩展集在线更新**：通过 GitHub 仓库 `Cx2bro-Extensions` 远程获取新扩展集，一键下载安装
- **PySide6 图形化界面**：全面重构，QTableWidget 表格展示、会社/作品颜色区分
- **扩展集索引系统**：`EXTENSIONS_INDEX.txt` + `generate_index.py` 自动化管理
- **底层能力**（C++）：
  - Detours DLL 注入引擎，运行时特征码扫描定位 TVP/Cxdec 接口
  - 运行时 Hash 采集（VTable Hook）+ 密钥提取（Hx/Cx/Verify 三探针）
  - 多线程并行还原 + 二次推理补充还原
  - 文本自动解密（XOR / 位交换 / zlib）
  - 批量 XP3 提取（Worker-Watchdog 架构）
